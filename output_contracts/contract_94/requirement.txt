This smart contract is a [Token Redeemer contract]. The main functions include [redeeming a specific token for another, redeeming multiple tokens at once, and setting redemption configurations]. The main states of the contract are [Redemption Configured, Redemption in Progress, and Redemption Completed]. The process logic for transitioning between states involves [setting up redemption configurations first, followed by executing the redemption of either a single or multiple tokens where specific validations and possible burning of tokens occur, ultimately leading to the issuance of new tokens]. The main variables include [super1155, burnAddress, customBurn, and redemptionConfigs]. The main functions include [redeem, redeemMult, and setRedemptionConfig]. The main events include [TokenRedemption and ConfigUpdate].
